# AmoCRM Deduplication Microservice

Микросервис для автоматического поиска и склейки дубликатов контактов в AmoCRM.
Разработан на FastAPI с использованием асинхронных запросов и упакован в Docker для быстрой интеграции и развертывания.

## Стек технологий

* **Python 3.13**
* **FastAPI** - высокопроизводительный веб-фреймворк для обработки вебхуков.
* **HTTPX** - асинхронный HTTP-клиент для работы с AmoCRM API v4.
* **Uvicorn** - ASGI-сервер.
* **Docker & Docker Compose** - контейнеризация приложения.
* **Ngrok** для локального тестирования.

## Ключевые возможности

* **Мгновенная реакция**: Принимает вебхуки от AmoCRM при создании нового контакта. Обработка логики вынесена в `BackgroundTasks`,
что позволяет моментально возвращать серверу 200 OK и избегать таймаутов.
* **Умный поиск дублей**:
     * По номеру телефона (автоматическая очистка от скобок, пробелов и спецсимволов перед поиском).
     * По Telegram-нику (реализован перехват ника из стандартного поля "Должность").
* **Безопасная склейка**:
     * Сохраняет самый старый контакт (Оригинал) как основной.
     * Дополняет Оригинал новыми данными.
     * Автоматически переносит Сделки и Примечания с дубликата на Оригинал.
* **Мягкое удаление**: Так как API v4 AmoCRM на уровне архитектуры запрещает прямое удаление контактов (ошибка 405 Method Not Allowed),
микросервис переименовывает обработанный дубликат в ДУБЛЬ (Склеен), позволяя менеджерам легко отфильтровать и удалить их вручную.
